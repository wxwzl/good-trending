# ADMEMS 架构设计方法论 Skill

基于温昱《一线架构师实践指南》的系统化软件架构设计方法。

## 概述

ADMEMS（Architecture Design Method for Enterprise Management Systems）是一种实用的架构设计方法体系，帮助架构师：

- 系统化地从需求推导出架构
- 识别关键需求作为架构驱动力
- 分阶段进行架构设计
- 处理复杂的非功能需求

## 核心内容

### 三阶段 + 一贯穿

```
Pre-Architecture → Conceptual Architecture → Refined Architecture
        ↓                   ↓                        ↓
   需求结构化          概念架构设计              细化架构(5视图)

              ↓↓↓ 贯穿环节 ↓↓↓
         非功能需求处理（目标-场景-决策表）
```

### Pre-Architecture 阶段

- **需求结构化** - ADMEMS矩阵方法
- **约束分析** - 识别约束及其对架构的影响
- **关键质量确定** - 5大原则确定质量目标
- **关键功能确定** - 4条规则识别核心功能

### Conceptual Architecture 阶段

- **初步设计** - 鲁棒图（边界-控制-实体）
- **高层分割** - 分层架构设计（Layer/Tier/通用性）
- **非功能考量** - 早期考虑质量属性

### Refined Architecture 阶段（5视图）

1. **逻辑架构** - 功能分解和模块结构
2. **物理架构** - 部署方案和拓扑结构
3. **运行架构** - 控制流和并发模型
4. **开发架构** - 项目组织和构建配置
5. **数据架构** - 数据组织和存储策略

### 非功能需求处理

- **场景技术** - 5要素描述质量场景
- **目标-场景-决策表** - 系统化处理非功能需求

## 关键概念速查

| 概念 | 说明 |
|------|------|
| ADMEMS矩阵 | 需求结构化工具，按功能/质量/约束 × 业务/用户/开发 分类 |
| 鲁棒图 | 初步设计工具，包含边界对象、控制对象、实体对象 |
| 5视图 | 逻辑、物理、运行、开发、数据 |
| 目标-场景-决策表 | 非功能需求处理工具 |

## 核心原则

1. **关键需求决定架构，其余需求验证架构**
2. **重大需求塑造概念架构**
3. **阶段是时间维度，视图是空间维度**
4. **质疑驱动的架构设计**
