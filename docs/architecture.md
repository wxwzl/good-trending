## 1.2 约束影响分析

```
约束: 全栈 TypeScript
├── 直接影响: 前后端、爬虫统一语言，类型共享
├── 间接影响: 可复用类型定义，减少接口文档维护
└── 隐含约束: 需要配置好 TypeScript monorepo 类型引用

约束: Monorepo + TurboRepo + pnpm workspace (5 apps + 4 packages)
├── 直接影响: 项目结构需要按 workspace 组织
├── 间接影响: 共享包管理方便，构建可缓存加速
└── 隐含约束: 需要正确配置 workspace 依赖关系

约束: Next.js 16 App Router（SEO核心）
├── 直接影响: 前端必须是 SSR/SSG 模式
├── 间接影响: 数据需要在构建时或请求时获取
└── 隐含约束: 需要考虑缓存策略和增量静态再生

约束: Playwright 爬虫 + Google 搜索
├── 直接影响: 爬虫需要无头浏览器环境
├── 间接影响: 资源消耗较大，需要反检测措施
└── 隐含约束: 需要处理 Google CAPTCHA 和反爬虫机制

约束: TDD 开发模式
├── 直接影响: 测试先行，代码可测试性要求高
├── 间接影响: 代码设计更模块化，易于重构
└── 隐含约束: 需要维护较高的测试覆盖率
```

# 架构设计总结

## 技术栈确认

| 层级       | 技术选型     | 版本  | 职责                   |
| ---------- | ------------ | ----- | ---------------------- |
| **前端**   | Next.js      | 16+   | SSR/SSG 渲染，SEO 优化 |
| **前端**   | React        | 19+   | UI 组件                |
| **前端**   | Tailwind CSS | 4+    | 样式方案               |
| **API**    | NestJS       | 10+   | RESTful API，业务逻辑  |
| **API**    | Swagger      | -     | API 文档               |
| **爬虫**   | Playwright   | 1.42+ | 数据采集，浏览器自动化 |
| **搜索**   | SerpAPI      | -     | Google 搜索结果 API    |
| **AI**     | 多提供商     | -     | Kimi/百炼/智谱 AI 分析 |
| **调度**   | BullMQ       | 5+    | 定时任务，队列管理     |
| **数据库** | PostgreSQL   | 16+   | 数据持久化             |
| **缓存**   | Redis        | 7+    | 缓存，队列存储         |
| **ORM**    | Drizzle ORM  | 0.44+ | 类型安全的数据库访问   |
| **构建**   | TurboRepo    | 2+    | Monorepo 构建缓存      |
| **包管理** | pnpm         | 9+    | 依赖管理               |
| **测试**   | Vitest       | -     | 单元/集成测试          |
| **E2E**    | Playwright   | -     | 端到端测试             |
| **国际化** | next-intl    | 4+    | 多语言支持             |
| **主题**   | next-themes  | 0.4+  | 明暗主题切换           |

## 核心设计决策

1. **Monorepo 结构**：5个 apps + 3个 packages，统一管理前后端和爬虫代码
2. **pnpm workspace**：通过 `workspace:*` 协议管理内部依赖
3. **Google搜索服务**：SerpAPI + 浏览器回退，增强反检测能力
4. **AI分析服务**：多提供商支持（百炼、Kimi、智谱），工厂模式切换
5. **Bitmap存储**：使用 BigInt 存储商品出现记录，高效计算趋势
6. **ISR 缓存策略**：平衡 SEO 和性能，减少服务器压力
7. **BullMQ 任务队列**：可靠的爬虫调度和重试机制
8. **TDD开发模式**：测试先行，确保代码质量和可测试性
